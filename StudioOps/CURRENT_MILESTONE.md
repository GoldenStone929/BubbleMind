## 用户反馈

- 编队与战斗需要五个角色槽位。
- 角色不能继续原地站桩；坦克应主动接近空间上最近的存活目标。
- 战斗节奏只需稍微放慢。
- 稀有度顺序统一为 `R -> SR -> SSR -> SP -> UR`。
- 星渊吞噬体是首位限定角色，定位为 `UR` 坦克。
- 当前模型的轮廓、黑洞层次与颜色明显偏离参考图，需要重制而非局部调色。
- 史莱姆进入攻击距离后必须留在前线持续攻击；仅当当前目标死亡时，才按当前位置重新寻找最近敌人。
- 角色需要数据化攻击距离与移动速度，不能继续用职业分支模拟射程。
- Unity 实机中的史莱姆颜色仍偏亮，黑洞核心不够明显，必须继续修正。

## 实施范围

1. 以单一五人队伍规则驱动存档、编队、战斗、UI、生成器和验证器。
2. 建立双方共用的五槽出生布局，并在模拟层保存每个单位的当前战场位置与持续锁定目标。
3. 为角色数据加入 `AttackRange` 与 `MoveSpeed`；每个固定逻辑 tick 先移动再判定攻击，进入射程后停留，目标死亡后按当前位置重新锁定最近敌人。
4. 将回放倍率由 `1.8` 调整为 `1.6`，不通过降低帧率制造慢速感。
5. 为角色数据加入正式限定标记，并把星渊吞噬体配置为 `UR`、限定、坦克及默认首槽。
6. 重做 Blender 角色主体比例、不对称角、眼睛、黑洞吸积盘、金属分段轨道、星点和近黑深靛紫材质，并保证黑洞在战斗三分之四视角可见。
7. 更新五槽 UI、自动验证、Play Mode 冒烟、Windows 构建与实机画面检查。

## 验收门槛

- 新存档默认拥有并保存五名不重复角色，Formation 与 Battle 均显示五槽。
- 玩家和敌方各生成五个单位；敌方阵容不复制限定角色。
- 星渊吞噬体在数据和 UI 中明确显示 `UR / TANK / LIMITED`。
- 坦克按当前战场位置选择最近存活敌人，进入自身攻击距离后留在原地持续攻击，不在每次命中后回到出生点。
- 当前目标存活时不换锁；目标死亡后按当前位置重选，并在新目标超出射程时继续移动。
- 受击、普攻和技能表现不得中断持续位移或把角色世界根节点重置到出生点。
- 五种稀有度顺序、文本与颜色均有验证覆盖。
- Blender 预览和 Unity 实机中的角色均呈近黑星空胶体、白紫眼睛、强黑洞核心与金属包边破碎环。
- 核心验证、Play Mode 冒烟、Windows 构建和运行日志全部通过。
