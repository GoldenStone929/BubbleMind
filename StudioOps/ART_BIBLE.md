# Art Bible

> 状态：预制作草案
> 当前用途：约束原创性、可读性和资产生产；不代表最终美术方向已经锁定。

## 当前视觉基线

当前试玩基线已切换为原创 2D 像素角色与像素地图，由 Unity 的正交深度、世界 UI、粒子、Shader 和 VFX 组成 2.5D 表现：圆润果冻轮廓、有限调色板、硬 Alpha、Point Filter、清晰像素边缘和克制发光。基础战斗单位统一为 Catherine 主角与元素史莱姆；旧 3D Prefab 和程序化人形仅保留为资产损坏时的兼容回退，不代表正常试玩画面。

## 永久视觉支柱

1. **原创且可识别**：角色、名称、符号、服装、武器、技能和世界观必须独立创作。
2. **轮廓优先**：移动端战斗距离下，职业、阵营、朝向和技能危险度必须一眼可读。
3. **系统一致**：模型、UI、VFX、动画与音频共享同一套色彩和强度层级。
4. **移动端可扩展**：每项正式资产在制作前定义面数、材质、贴图、骨骼、动画、LOD 和内存预算。
5. **资产可追溯**：任何生成、购买或外包内容必须记录来源、权属、许可证和人工修改过程。

## 首个正式样板方向

`ART-CHAR-PIXEL-PARTY-001` 是当前运行时角色样板；Catherine 使用近黑深靛紫像素凝胶、内部星空、纯黑事件视界、白紫吸积盘、不对称角和破碎紫金轨道，四只默认伙伴用土、火、风、雷的独立轮廓识别。`ART-CHAR-UR-COSMIC-SLIME-001` 的 3D 版本保留为历史造型与兼容参考，不再是默认战斗表现。

样板原则：

- 轮廓在关闭半透明、发光和粒子后仍清晰可读。
- 战斗 Sprite 统一为 128×128 RGBA、PPU 64、底部中心 Pivot、Point Filter、无 Mipmap、Clamp 和无压缩；复杂星空、黑洞和吸积盘可继续由 Unity Shader/VFX 承担。
- 史莱姆共享有限调色板、眼睛比例、像素密度和高光语言，禁止混入 Bilinear 模糊或不同像素网格。
- 软体生命感以 `Squash & Stretch` 为核心；移动、攻击、受击和大招先由像素级位移与局部缩放表达，后续升级为等效逐帧 Sprite Sheet。
- UR 稀有度通过独特轮廓、材质层次和演出强度表达，不只依赖紫色。
- Catherine 的轨道环必须保持独立可读层，可由 Sprite、Shader 或 VFX 单独驱动，不能与主体烘成不可维护的单体。
- 免费 Tripo 产物只作为非商用内部原型；商业资产权利必须另行闭环。

## 首批战斗地图方向

地图共享清晰的左右五人出生阵列；当前 5v5 显示十个运行时标记。战场主轴按 20 格校准，并保留无遮挡中央战区、低频像素地面纹理和不遮挡世界血条的远景构图。中央区域必须给单位从出生点走向各自射程位置留出清晰通道。当前固定正交镜头使用 480×270 Point Filter 像素背板，配合 Unity 深度站位、世界 UI、粒子与技能 VFX；未来扩展自由镜头时再升级为分层 Tilemap 或模块化场景。

1. **星渊观测台 / Abyssal Observatory**：悬浮于星海的古代圆形观测场，柔和靛灰石材、暖金描边、冷青晶体与紫白奇点。远方奇点只承担景深焦点，战斗地面保持低频中性，避免角色融入背景。本轮实现。
2. **苍穹铸庭 / Verdant Skyforge**：高空锻造遗迹与悬浮花园，冷灰石、氧化青铜、翠绿植被和明亮天光；强调清爽可读与昼间地图差异。仅设计，不生成资产。
3. **镜潮陨湾 / Glass-Tide Crater**：月下陨石盆地与黑色玻璃潮汐，银灰岩层、青白反射和少量珊瑚红警示光；强调低反射地面与远景水面层次。仅设计，不生成资产。

首版地图禁止在十个出生标记和主要前线路径后方放置高亮星点、细柱或复杂纹样；地图不得成为角色轮廓和战斗信息的竞争焦点。

## 已锁定的角色族方向

- 标准测试单位采用圆润、低重心、无四肢或极简附肢的基础史莱姆比例。
- 水、火、土、风、雷通过外轮廓与元素装饰区分；同族共享眼睛、表情、体积和动作节奏。
- 限定 UR 允许显著增加轮廓、材质层次与独立演出；星渊吞噬体使用近黑胶体、纯黑事件视界、白紫吸积结构和紫金轨道，与标准单位建立明确稀有度差异。
- 目标平台优先级仍待后续锁定；本轮继续以 Windows 试玩和移动端可扩展预算作为共同门槛。

## 色彩、材质与信息层级

- 阵营/职业色不得成为唯一识别手段；同时使用轮廓、武器、姿态和图形符号。
- 可交互 UI、冷却、危险提示、治疗、伤害和稀有度使用稳定且互不冲突的语义。
- 材质数量、透明层、屏幕空间特效和发光强度必须受移动端性能预算约束。
- 任何参考图只用于风格分析，不复制现有 IP 的角色、构图、标志性服装或特效。

## 动画与 VFX 语言

- 动画事件必须与战斗模拟事件解耦；表现可以延迟，但不得改变确定性结果。
- 普攻、技能、受击、治疗、倒地和胜利需要清晰的预备、命中和恢复阶段。
- 普攻、技能和受击只恢复模型局部姿态，不得把角色的逻辑世界位置重置到出生点。
- VFX 强度遵循：普通攻击 < 小技能 < 核心技能 < 战斗终结；不得持续遮挡角色状态和 UI。
- 黑洞类终结技必须有清晰的蓄力、吸附、持续命中、坍缩和余波五段节奏；每一段的颜色、尺度和屏幕占比都要服从角色与血条可读性。
- 付费 VFX/Shader 包只有在明确列出商品、当前价格、版本、渲染管线、许可和目标路径并获得用户单独确认后才能购买或接入；未购买时可以参考功能覆盖，但不得复制素材。

## 正式资产进入 Unity 前的门禁

- 已有独立 `Asset Spec`。
- 来源、许可证和权属已写入 `ASSET_LEDGER.csv`。
- 命名、比例、坐标、骨架、材质、贴图和 LOD 符合规格。
- 在目标 Render Pipeline 与代表性设备上验证。
- 导入设置、Prefab 契约和性能数据有证据可复查。

## 禁止项

- 不使用或近似复刻 Jujutsu Kaisen、Pokémon 或其他既有 IP 的角色、名称、标志、世界观与招牌表现。
- 不从 XAPK、反编译内容或来源不明的模型包提取资产。
- 不在未确认训练/商用条款、输出权利和素材来源时引入生成式资产服务。
