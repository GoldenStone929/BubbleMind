# BubbleMind 清洁室知识库

> 建档日期：2026-07-15
> 范围：只读研究、通用系统归纳、BubbleMind 原创映射
> 运行时边界：本目录只服务于设计与审计，游戏代码和内容不得反向依赖参考证据

## 目的

本知识库把三类容易混在一起的信息强制分开：看见或读到的事实、从事实归纳出的通用规律，以及 BubbleMind 独立创作的实现决定。它不是外部游戏复刻说明，也不是外部角色、技能、数值或美术资源的存档。

当前结构快照：22 条证据、36 个系统模块、26 个页面、27 个数据实体和 90 条唯一关系；所有系统、页面与实体节点都至少拥有一条可解析关系边。

## 三层结构

### 第一层：证据与覆盖

- `EVIDENCE_LEDGER.csv`：证据来源、保留策略、观察状态与可支持的声明范围。
- `COVERAGE_MATRIX.csv`：36 个中性系统模块的资料、实时窗口与用户截图覆盖，含置信度和修改风险。
- 该层只回答“证据是什么、能证明什么、还缺什么”。

### 第二层：通用系统知识

- `REFERENCE_SYSTEM_ATLAS.md`：模块族、页面族、数据实体、全流程、状态权威与已知/未知边界。
- `RELATIONSHIPS.csv`：页面、系统与实体之间的中性关系图；当前覆盖全部 36 个系统、26 个页面和 27 个实体。
- `GENERALIZED_PATTERNS.md`：从证据归纳出的可复用产品和架构规律。
- 该层只回答“一个同类产品通常怎样组织”，不保留外部视觉身份或专有表达。

### 第三层：BubbleMind 原创落地

- `BUBBLEMIND_MAPPING.md`：通用模块到当前原创 Unity 工程的映射、状态和后续接口。
- `ORIGINALITY_CHECKLIST.md`：研究、设计、内容、代码和发布前的原创性检查。
- 该层只回答“BubbleMind 自己怎样做”，所有名称、数值、文案、角色、美术和代码必须独立创作。

## 声明标签

文档采用以下语义；不得把较弱标签升级为事实：

| 标签 | 含义 |
|---|---|
| `FACT` | 用户截图、只读实时观察或项目文件可直接支持的事实 |
| `ANALYSIS` | `analysis/architecture/` 已记录的清洁室架构结论；仍不等于外部实现细节已被恢复 |
| `INFERENCE` | 多项证据支持的通用推断，需要原创验证 |
| `UNKNOWN` | 当前没有足够证据，禁止用常识或想象补齐 |
| `BUBBLEMIND` | 本项目独立设计或当前实现，不是外部事实 |

## 覆盖与置信度

`COVERAGE_MATRIX.csv` 使用固定枚举：

- `analysis_coverage`：`documented`、`partial`、`none`
- `live_coverage`：`observed`、`partial`、`blocked`、`not_observed`
- `screenshot_coverage`：`observed`、`partial`、`none`
- `confidence`：`C0` 无证据、`C1` 仅弱线索、`C2` 有架构证据、`C3` 多源印证、`C4` 直接观察且有架构交叉验证
- `mutation_risk`：`none`、`low`、`medium`、`high`、`critical`

置信度表示“我们对系统边界的了解程度”，不表示能复制外部规则。即使为 `C4`，外部精确数值、布局坐标、角色内容、代码和协议仍然在禁止范围内。

## 强制边界

1. `analysis/` 只读；不得修改、移动或把其中生成物复制进运行时。
2. 不读取、运行、解包、解密或提交工作区根目录的 XAPK/APK。
3. 参考游戏只做只读观察；不得抽卡、消费、领取、养成、改队、发送消息、保存设置、进入战斗、迁移基地、发起世界行动、切换账号或关闭窗口。
4. 外部截图不进入仓库；台账只保留中性描述和“不保留”状态。
5. 不记录外部角色实名、长篇专有文案、技能原文、精确概率、精确数值、布局坐标、图标、字体、资源、接口、协议或凭据。
6. 页面相似性只能用于信息层级和交互职责归纳，不能用于像素级、构图级或品牌身份复刻。
7. 正式经济、抽取、奖励、账号、支付、竞技与排行榜必须由后端权威；本地 Demo 不能被描述成生产方案。
8. 新结论必须先进入证据台账和覆盖矩阵，再进入通用规律；原创实现不得以未标记的推断作为需求。

## 当前实时观察状态

`FACT`：已只读观察首页、角色列表、角色详情、技能说明、招募、概率说明、独立大世界，以及样本角色的领域、命运和境界子页。省电遮罩后已恢复观察，但用户需要使用电脑时立即停止全部游戏操作。窗口保持开启，没有重启、退出、重新登录或执行账号状态操作。精确截图和专有角色信息只保存在本机忽略目录，知识库只登记中性系统结论。（证据：`LV-007`、`LV-008`）

## 更新流程

1. 登记新证据和保留策略。
2. 更新对应模块覆盖，不越级提高置信度。
3. 在关系表中补充中性连接。
4. 只把跨证据稳定成立的结论写入通用规律。
5. 通过原创性检查后，才允许更新 BubbleMind 映射或实现。
