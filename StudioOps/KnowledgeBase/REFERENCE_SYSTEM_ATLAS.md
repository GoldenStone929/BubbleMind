## 5. 完整产品流程

### 5.1 启动到首页

```text
P01 启动/更新
  -> M01 初始化日志、平台抽象和本地配置
  -> M02 校验 E25 内容目录与必需内容
  -> M03 建立 E01 会话
  -> 可选 P03 选择服务区
  -> 加载 E02 玩家档案和后端状态快照
  -> P04 首页
```

`UNKNOWN`：真实握手、消息编号、压缩加密、服务路由和下载签名均未恢复，也不得复用。

### 5.2 抽取到角色收藏

```text
P15 抽取页读取 E12 横幅与展示成本
  -> P16 只读显示合规说明
  -> 后端校验时间、E14 余额、E13 池与 E22 计数
  -> 返回权威结果和状态增量
  -> P17 只负责揭示
  -> 新角色写入 E04；重复结果按原创规则转为 E15
  -> P06 角色收藏刷新
```

客户端不得决定随机结果、扣费、保底或重复转化。`UNKNOWN`：外部池内容、概率、保底和商品映射。

### 5.3 角色养成与编队

```text
P06 选择 E04
  -> P07 合并 E03 静态定义与 E04 动态状态
  -> P10 读取 E05/E06
  -> P08 请求养成或 P09 请求装备变更
  -> 后端校验 E14/E15/E08
  -> 返回更新后的 E04
  -> P11 把合法实例写入 E09
```

`UNKNOWN`：外部成长公式、材料成本、解锁要求、装备词条与角色专属规则。

### 5.4 关卡、战斗与奖励

```text
P12 选择 E10
  -> P11 校验 E09
  -> 创建 E19（模式、种子、队伍、敌人、规则版本）
  -> P13 确定性模拟：目标、移动、技能、状态、伤害、死亡
  -> 产生 E20
  -> 后端校验奖励资格
  -> 应用 E21 到 E14/E15/E16/E04
  -> P14 展示结果
  -> 返回 P12 或 P04
```

`UNKNOWN`：外部伤害公式、AI、时间线、波次、统计签名和奖励值。

### 5.5 日常与活动循环

```text
后端 E26 时间/重置状态
  -> M23 过滤 E17 活动窗口
  -> M20/M22/M24 更新任务、日历和邮件
  -> E27 派生首页提醒
  -> 玩家只读查看或显式领取
  -> 后端应用 E21
```

客户端本地时间只用于显示提示，不能决定资格、重置或奖励。

### 5.6 内容更新与角色资产

```text
内容作者创建 E03/E05/E06 与原创模型、卡面、VFX、SFX
  -> M36 校验骨架/动作/插槽/预算/引用
  -> 构建 E25 目录与哈希
  -> M02 签名分发
  -> UI 与战斗按稳定身份键异步加载
```

下载内容不得包含未经签名的可执行脚本；参考资产和截图不得进入该流程。

## 6. 状态权威

| 状态类别 | 权威来源 | 客户端允许 | 客户端禁止 |
|---|---|---|---|
| 账号、会话、玩家档案 | 后端 | 缓存和显示 | 自行签发身份或改变权限 |
| 货币、物品、装备实例 | 后端 | 预览、提交意图、应用返回增量 | 本地直接授予或扣除正式资产 |
| 抽取、保底、重复转化 | 后端 | 显示配置和揭示返回结果 | 本地权威随机、改池或改计数 |
| 购买与支付 | 平台支付加后端 | 发起订单并显示状态 | 根据客户端回调直接发货 |
| 任务、活动、邮件、奖励 | 后端时钟与后端状态 | 显示、导航、领取请求 | 用本地时间决定资格或重复领取 |
| PVP、排行、竞争奖励 | 后端 | 展示和提交输入 | 信任客户端结果或排行写入 |
| 角色、技能、关卡静态定义 | 经验证的内容目录 | 缓存、加载、版本检查 | 使用未登记或未签名内容 |
| 战斗模拟 | 确定性核心；正式奖励由后端复核 | 本地预测、重播和表现 | 让动画帧或 VFX 反向改写结果 |
| UI 导航、筛选、展开状态 | 客户端 | 本地持久化非敏感偏好 | 将其当作经济或进度权威 |
| 红点 | 客户端派生显示 | 从权威状态重算 | 成为独立业务真相来源 |

## 7. 已知、推断与未知

### 已知

- 存在启动、内容分发、登录、首页、角色、装备、技能、编队、战斗、抽取、经济、运营、社交和表现等模块边界。
- 页面组织至少支持首页到角色、招募、关卡、竞技、活动、商店、邮件、设置和社交入口。
- 角色、技能、编队、关卡、抽取池、货币、物品、任务、活动、商店与战斗结果可用数据驱动实体表达。
- 直接观察页面重视场景语境、角色图像识别、列表与详情分工、池主题与概率入口。

### 推断

- 首页是订阅玩家快照、开放条件、服务器时间和红点派生状态的功能路由器。
- 角色详情把静态定义、玩家实例、装备和技能状态组合成读模型。
- 技能模拟应与动画、VFX 和音频表现分离，并由稳定时间线事件连接。
- 正式抽取、支付、奖励和竞争结果应后端权威；本地实现只能是清晰标注的测试替身。

### 未知

- 未直接观察页面的真实结构、输入反馈、加载态、错误态和可访问性行为。
- 外部角色全量目录、技能内容、成长曲线、战斗公式、AI、波次和动画事件。
- 外部概率、保底、重复转化、商品、货币、奖励、重置和开放条件。
- 网络协议、消息结构、端点、认证、内容加密和服务端验证方式。
- 外部骨架、模型、材质、动画、VFX、音频和资源依赖图。
