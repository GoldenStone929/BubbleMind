# BubbleMind 原创性检查表

> 用法：新增角色、技能、页面、系统、资产或里程碑时逐项检查。任一“停止条件”触发时，不得进入实现或提交。

## A. 研究输入

- [ ] 证据已登记在 `EVIDENCE_LEDGER.csv`，包含来源、日期、保留策略和声明范围。
- [ ] `analysis/` 仅只读，没有修改、移动或复制生成物到运行时。
- [ ] 没有读取、运行、解包、解密或提交 XAPK/APK。
- [ ] 外部截图未复制进项目，仓库只保留中性观察。
- [ ] 参考游戏窗口没有被关闭、重启、退出登录或切换账号。
- [ ] 没有抽取、消费、领取、养成、改队、开战、发送、保存设置或执行其他账号状态操作。
- [ ] 未观察页面明确标记为 `UNKNOWN`，没有用常识补齐。
- [ ] 实时窗口若受遮罩、登录、支付或不可逆流程阻挡，已停止而不是尝试绕过。

## B. 事实与推断

- [ ] 每个结论可以标为 `FACT`、`ANALYSIS`、`INFERENCE`、`UNKNOWN` 或 `BUBBLEMIND`。
- [ ] 截图只支持画面中直接可见的信息，不被扩大为系统实现事实。
- [ ] 架构资料只支持模块边界和通用设计，不被描述成外部代码已恢复。
- [ ] 置信度按 `C0-C4` 规则填写，没有因为“看起来合理”而升级。
- [ ] 精确公式、数值、概率、协议、资源和角色内容在无原创来源时保持未知。
- [ ] 新关系已进入 `RELATIONSHIPS.csv`，且标明权威与确定性。

## C. 产品设计

- [ ] 页面职责来自 BubbleMind 玩家流程，而不是为了匹配外部页面数量。
- [ ] 布局、构图、坐标、图标、色彩、字体、边框和动效是独立设计。
- [ ] 文案是原创并与 BubbleMind 世界观一致，没有改写外部长篇专有文案。
- [ ] 信息架构只使用通用规律，例如列表/详情分工、稳定身份键和明确状态。
- [ ] 首页入口分组、角色详情分栏和玩法目录的名称、顺序、视觉层级与交互均按 BubbleMind 自身流程重新设计。
- [ ] 已定义加载、成功、空、锁定、失败和不可操作状态中适用的部分。
- [ ] 主操作、返回路径、资源变化和错误反馈清楚。
- [ ] 不使用外部品牌、角色实名、术语、徽记、服装、武器或标志性视觉身份。

## D. 角色与技能内容

- [ ] `characterId` 在项目中唯一、稳定且采用 BubbleMind 命名。
- [ ] 显示名、称号、阵营、简介、职业组合和元素身份为原创。
- [ ] 卡面、头像、模型、材质、纹理、动作、VFX、SFX 和声音均有原创或许可来源。
- [ ] 技能名称、机制组合、目标规则、文案、公式、数值和时间线为原创。
- [ ] 能力通过通用数据字段表达，没有为外部角色写专属复制逻辑。
- [ ] 静态角色定义与玩家拥有实例分离。
- [ ] 静态目录和持有阵容共享角色定义，但不会把目录可见性、拥有状态或重复实例互相混淆。
- [ ] 技能定义、目标规则、时间线和表现事件分层。
- [ ] 角色档案的基础攻击、运行时技能槽、扩展能力、养成和获取记录完整。
- [ ] 内容审批、来源说明和允许公开状态已填写。

## E. 美术与音频资产

- [ ] 未使用外部截图作直接图生图输入，除非用户拥有并明确授权且台账已登记。
- [ ] 未追踪、描摹或复刻外部角色轮廓、服装、姿势、构图或标志性特效。
- [ ] 生成式资产提示词不含外部角色实名或要求模仿在世艺术家。
- [ ] 第三方字体、音频、纹理、模型、Shader 和工具都有许可证、来源、版本和校验记录。
- [ ] 资产台账与工具依赖台账没有混用。
- [ ] 模型骨架、动作、插槽、材质数、纹理尺寸、透明度和粒子预算通过项目门槛。
- [ ] 缺失资源使用 BubbleMind 原创回退，不引用参考资源。

## F. 数据与代码

- [ ] 运行时代码不读取 `analysis/`、知识库、外部截图或参考路径。
- [ ] 页面不维护角色、概率、余额、拥有状态或技能的第二份权威数据。
- [ ] 正式账号、货币、物品、装备、抽取、保底、支付、奖励、活动、邮件、PVP 和排行由后端权威。
- [ ] 本地 Mock 明确标记为 Demo/Test，不能被误用为生产安全方案。
- [ ] 请求接口预留幂等键、内容版本、会话身份和错误恢复。
- [ ] 战斗模拟固定 Tick、可种子化、可重播；表现层只消费事件。
- [ ] 规则预览、编成校验、战斗上下文和奖励资格引用同一版本化玩法模式，周期资格不由本地文案或时钟决定。
- [ ] 新角色和技能通过数据增加，不要求修改通用战斗核心。
- [ ] 所有引用、ID、枚举、范围和内容审批状态可自动验证。

## G. UI 与可访问性

- [ ] 同一角色在角色页、抽取、编成和战斗入口显示同源身份与卡面。
- [ ] 角色详情切换分栏时保持选中角色、实例与返回路径；首页入口组切换时保持全局壳和当前位置提示。
- [ ] 已拥有、未拥有、已选择、锁定、加载和不可操作状态不仅依靠颜色区分。
- [ ] 文本在目标桌面和移动比例不溢出、不重叠、不遮挡主要内容。
- [ ] 图标按钮有可理解的图形和工具提示，命令按钮层级明确。
- [ ] 页面支持键盘、手柄或触控目标中的预定输入方式。
- [ ] 动效、闪烁、镜头震动、音量和速度有可降级策略。
- [ ] 列表规模扩大后仍使用复用、分页或虚拟化方案。

## H. 验证与发布

- [ ] 编辑器内容验证通过：重复 ID、缺图、断链、非法枚举、未审批来源均会失败。
- [ ] 确定性测试证明同一上下文和种子产生相同结果。
- [ ] 页面流程测试覆盖首页、角色、抽取、编成、战斗和结果。
- [ ] 真实窗口检查目标分辨率、字体、裁切、遮挡和交互反馈。
- [ ] 构建日志无编译、Shader、丢资源或许可证错误。
- [ ] 新依赖和资产已分别登记版本、来源、许可证、路径和校验值。
- [ ] 成功更新已写入里程碑或日志，并说明验证证据和剩余限制。
- [ ] 提交只包含本项目原创或合法许可内容，不含外部截图、缓存、构建、凭据或分析生成物。

## 停止条件

出现下列任一情况，立即停止该内容进入实现或发布：

1. 来源不明、授权不明或无法证明原创。
2. 需要复制外部角色、技能文案、精确数值、概率、布局、代码、协议或资源才能完成。
3. 需要操作参考账号的支付、抽取、领取、养成、编成、社交、设置或登录状态才能补证。
4. 需要读取、运行、解包、解密或提交 XAPK/APK。
5. 本地客户端被要求成为正式经济、支付、奖励或竞争结果的唯一权威。
6. 证据与实现来源无法分离，或运行时开始依赖研究资料。

停止后应把事项记录为 `UNKNOWN` 或 `blocked`，给出原创替代方案，再继续其他不受阻工作。
