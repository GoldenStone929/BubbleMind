# StudioOps

`StudioOps` 是本项目的轻量制作管理层。它吸收大型游戏工作室流程中真正有用的部分，但不复制 49 个角色、Shell Hooks 或重复的路线图。

## 信息权威映射

| 内容 | 唯一权威 |
|---|---|
| 长期范围、技术架构、安全边界、阶段与决策 | `../PROJECT_PLAN.md` |
| 当前正在执行的工作包 | `CURRENT_MILESTONE.md` |
| 运行时架构 | `../Assets/_Game/Docs/ArchitectureSummary.md` |
| 美术方向与技术美术约束 | `ART_BIBLE.md` |
| 模型、贴图、动画、音频等内容资产 | `ASSET_LEDGER.csv` 与按需创建的 Specs |
| 工具、插件、包与许可证 | `../Assets/_Game/Docs/ThirdPartyInventory.md` |
| 验证结论 | `../Assets/_Game/Docs/VerificationReport.md` 与 `../Artifacts/` |

同一信息只维护一次；其他文档使用链接，不复制整段内容。

## 精简职责

| 职责 | 何时启用 | 主要产出 |
|---|---|---|
| 制作与范围 | 每个里程碑 | 优先级、范围、风险、交接 |
| 游戏设计与数据 | 新玩法或平衡 | 规则、数据契约、验收案例 |
| Unity 架构与玩法 | 系统实现 | 可测试代码、场景、Prefab、迁移说明 |
| UI/UX | 玩家流程变化 | 交互、状态、响应式与无障碍要求 |
| 美术与技术美术 | 正式资产生产 | Art Bible、资产规格、导入与性能预算 |
| 动画/VFX/音频 | 表现阶段 | 时间线、事件契约、资源与混音规范 |
| QA | 每次交付 | 测试矩阵、缺陷证据、回归结果 |
| 发布与依赖 | 包、构建或平台变化 | 依赖清单、许可证、构建与发布清单 |

职责按任务启用，不创建常驻的虚拟岗位文件。

## 工作循环

1. 从 `PROJECT_PLAN.md` 选择一个已批准目标。
2. 在 `CURRENT_MILESTONE.md` 写清范围、非目标、交付物与验收门槛。
3. 先建立可回滚基线，再实施最小完整切片。
4. 运行与风险相称的验证并保存证据。
5. 更新主计划决策、资产/依赖台账和里程碑状态。
6. 用一个清晰提交完成交接；远程发布必须另行授权。
