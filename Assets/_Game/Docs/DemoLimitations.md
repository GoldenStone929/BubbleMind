# Demo Limitations

- 这是离线玩法原型，不是生产游戏或联网服务。
- 存档、货币和抽卡均在本机执行，不具备服务器权威性或防作弊能力。
- 当前七名角色已有独立像素战斗 Sprite；六名标准角色仍从五元素基础设定衍生，水系两名角色共享同一基础轮廓，尚未达到最终角色差异化程度。旧 Blender 模型只保留为兼容回退。
- 当前只有单抽；重复角色会转换为 Universal Shard，但十连、保底、碎片消耗/觉醒、角色升级和完整养成仍属于后续阶段。
- 当前只制作 Chapter 01 的三关离线主线，关卡共用同一战斗场景与确定性五人测试敌队；尚无章节选择、分支路线、动态敌群或独立 Boss 行为包。
- 没有登录、网络、IAP、广告、实时活动、远程内容、线上 PvP 或正式音频。Arena、Events、Shop、Mail、Guild 只有明确的锁定说明页，不是可用在线系统。
- 当前移动使用固定 Tick 的直线接近，没有 NavMesh、碰撞绕行或复杂障碍寻路；20 格战场内 Tank / Assassin 统一射程 2，其余职业统一射程 10，尚未提供每个关卡的动态射程修正。
- 玩家保存的合法五槽阵容会直接进入本地 5v5 战斗；目前敌方仍是确定性测试阵容，尚无真实玩家匹配、联网同步或服务器裁决。
- 当前怒气固定为 0–1000、普攻命中 +100、受伤 +50；技能槽 2 / 3 使用 5 秒错峰的固定 10 秒自动时间表，尚未实现手动大招、沉默、打断、施法队列或复杂技能 AI。
- 战斗表现以清晰可读为目标；多人近战和黑洞聚拢时仍可能发生 Sprite 或姓名牌相交，部分动作和事件重播可在下一轮继续打磨同步感。
- 基础史莱姆目前使用单帧像素 Sprite 配合程序化位移、Squash & Stretch、闪白和现代 VFX；尚未制作逐帧表情、行走、攻击和倒地 Sprite Sheet。
- Catherine 当前使用 128×128 像素 Sprite 加黑洞 Shader/VFX；正式商业角色仍需要完整逐帧动画、移动端透明过绘优化和参考图商业权利确认。旧 19,000 面凝胶模型只保留为内部兼容原型。
- Settings 会保存音量、全屏与 30/60 FPS 选择，并提供本地数据重置确认；由于当前没有正式音乐/音效，音量滑杆主要验证状态与运行时接口。
- 鼠标、触摸、键盘和基础手柄选择可用，但尚未制作完整的无障碍、按键重映射、语言或画质等级页面。
- Windows Build 与 Unity Editor 版本共享相同的本地 Demo 逻辑。
